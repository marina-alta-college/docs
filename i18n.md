# Internationalisation (i18n)

The site supports 4 languages: English, Spanish, German, and Dutch.

## Two-Layer System

### 1. UI Strings (locale JSON files)

Static text like navigation labels, button text, and error messages. These live in `site/locales/`:

```
locales/
├── en.json    # English (source of truth)
├── es.json    # Spanish
├── de.json    # German
└── nl.json    # Dutch
```

Used in templates with:
```html
{{t .T "nav_admissions"}}
```

The `T()` function falls back to English if a key is missing in the target language, then returns the key itself as a last resort.

### 2. CMS Content (Sanity field variants)

All Sanity content fields have per-language variants:

| Field | English | Spanish | German | Dutch |
|---|---|---|---|---|
| Title | `title` | `title_es` | `title_de` | `title_nl` |
| Body | `body` | `body_es` | `body_de` | `body_nl` |

The Go server uses `coalesce()` in GROQ queries to fall back to English:

```groq
"title": coalesce(title_es, title)
```

## Language Detection

When a user visits `/` (root), the server detects their language:

1. Check `lang` cookie (set by previous visit or language switcher)
2. Parse `Accept-Language` header
3. Default to `en`

Then redirects to `/{lang}`.

## URL Structure

```
/en/admissions     → English admissions page
/es/admissions     → Spanish admissions page
/de/admissions     → German admissions page
/nl/admissions     → Dutch admissions page
```

The language switcher (`/set-lang/{lang}`) sets a cookie and redirects to the same page in the new language.

## Adding a New UI String

1. Add the key to `locales/en.json`
2. Add translations to `es.json`, `de.json`, `nl.json`
3. Use `{{t .T "your_key"}}` in templates

## Adding a New Language

This requires changes in multiple places:

1. **`i18n.go`**: Add to `supportedLangs` and `langMetas`
2. **`sanity.go`**: Add to `langSuffix()` switch statement
3. **`locales/`**: Create new `{lang}.json` file
4. **Sanity schemas**: Add new `localizedString`/`localizedBlockContent` variants (update the `LANGS` constant in each schema file)
5. **`render.go`**: Add month names and date formatting for the new language
